---
name: Quality Expectations
description: User expects polished, production-quality UI without being asked — not basic/minimal implementations
type: feedback
---

Always build polished, production-quality UI from the start. Don't deliver basic/minimal pages expecting the user to ask for improvements.

**Why:** User was frustrated that auth pages were basic and they had to ask for better ones. They expect premium quality by default.
**How to apply:** When building UI pages, always include: split-layout designs for auth, animations, gradient backgrounds, testimonials/social proof, feature highlights, password strength indicators, proper visual hierarchy, and attention to detail. Don't ship "functional but plain" — ship beautiful.
